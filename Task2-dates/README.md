# Отчёт

## Деплой

- Для того чтобы развернуть проект достаточно выполнить 

```bash
docker compose up -d --build
```
- Для работы через nginx-proxy требуется настроить .env файл (Пример оформления .env файла можно найти в .env.example)

## Backend

1. Бэкед сервиса теперь использует gunicorn
2. Для оптимизации выбран image python11-slim с последующей установкой требующихся системных зависимостей
3. Используется pip wheel для более быстрой и стабильной установки
4. Сервис запускается от пользователя appuser, который не имеет root прав 

## Frontend

1. Установка пакетов через pnpm
2. Разделение логики внутри докер файла на 3 этапа
3. Сервис запускается от пользователя nextjs, который не имеет root прав  

## Особенности

1. В виду особенностей работы nginx-proxy не удалось добавить возможность разворачивать проект без доменного имении с доступом через http://localhost:80
